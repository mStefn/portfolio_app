🏋️‍♂️ Gym Tracker: PWA & Full-Stack DevOps Architecture

A self-hosted Progressive Web App (PWA) designed for tracking gym workouts. While the application provides a clean and intuitive user interface, the primary focus of this project is the production-ready infrastructure and advanced DevOps practices running under the hood.

Live Application

The system is fully containerized using Docker and is logically split into two independent environments: the Application Layer and the Infrastructure Layer. This separation reflects real-world system design, improving both scalability and maintainability.

Architecture & Technology Stack

  • Application Layer: A high-performance Backend API written in Go, an Nginx-served Vanilla JS Frontend, and a MariaDB database.
  • Infrastructure & Observability: A robust monitoring stack utilizing Prometheus, Loki, and Grafana Alloy. Dashboards are managed as code in Grafana to provide comprehensive visibility.
  • Code Quality: Integrated SonarQube (backed by PostgreSQL) ensures strict code quality checks before any deployment.
  • Data Persistence: Stateful data is secured using Host Bind Mounts, guaranteeing reliable backups across all databases and monitoring states.

CI/CD & Secure Deployment

The deployment pipeline is fully automated via GitHub Actions. Docker images are built and pushed to the GitHub Container Registry.

Crucially, the deployment process executes over a private Tailscale VPN. This zero-trust security model means there is no public SSH exposure. Additionally, the pipeline handles real-time deployment notifications via Discord and performs post-deployment health checks to ensure system stability.

Project Purpose & Future Scope

This project was built to demonstrate practical DevOps skills, real-world deployment workflows, and secure infrastructure design. In the future, the infrastructure will be further automated using Ansible for server provisioning and Terraform for Infrastructure as Code (IaC).
