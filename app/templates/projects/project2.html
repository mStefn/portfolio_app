🏋️‍♂️ Gym Tracker: Go, PostgreSQL & Docker

A comprehensive workout tracking application designed to monitor gym progress, manage exercise routines, and analyze performance over time.

The project is self-hosted on an on-premise Ubuntu Home Lab and focuses on efficient containerization and a lightweight backend architecture:

  • Backend: Built with Go (Golang) using standard libraries for high performance and low memory footprint.
  • Database: PostgreSQL used for reliable and structured data storage of workout logs and user metrics.
  • Infrastructure: Fully containerized using Docker and Docker Compose for seamless deployment on my home server.

This setup allows me to maintain full control over my data while experimenting with bare-metal server management and container orchestration.

Key Code Fragments & Explanations

1. Multi-Stage Dockerfile for Go Application

FROM golang:1.21 AS builder
WORKDIR /app
COPY go.mod go.sum ./
RUN go mod tidy
COPY . .
RUN CGO_ENABLED=0 GOOS=linux go build -o gym_app ./main.go

FROM debian:bookworm-slim
WORKDIR /app
COPY --from=builder /app/gym_app ./
CMD ["./gym_app"]

Why it matters: Using a multi-stage build significantly reduces the final image size. The app is compiled in a heavy Go environment, but runs in a stripped-down Debian container, saving resources on the home lab.

2. Go HTTP Handler for Saving Workouts

func saveWorkout(w http.ResponseWriter, r *http.Request) {
    if r.Method != http.MethodPost {
        http.Error(w, "Method not allowed", http.StatusMethodNotAllowed)
        return
    }
    // Parse form and insert into PostgreSQL
    err := insertWorkoutToDB(r.FormValue("exercise"), r.FormValue("reps"))
    if err != nil {
        http.Error(w, "Database error", http.StatusInternalServerError)
        return
    }
    w.WriteHeader(http.StatusCreated)
}

Why it matters: Demonstrates clean and straightforward routing using Go's standard net/http library without relying on heavy external frameworks.

Infrastructure and Workflow Summary

  • Self-hosted on a bare-metal Ubuntu server.
  • Environment managed via docker-compose for easy environment replication.
  • Persistent volume mapping for the PostgreSQL database to ensure no data loss during container restarts.

This project combines software engineering with home-lab system administration, creating a practical, daily-use application.
